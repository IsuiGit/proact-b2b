# SCOUT Scorer — правила ранжирования сырых событий

## Входные данные
`scout_raw.json` — массив объектов с полями:
- `id`, `company_name`, `inn`, `source`, `event_type`, `event_description`, `amount` (в рублях), `detected_at` (ISO), `company_id`
- `event_type` может быть: `new_procurement`, `capital_change`, `court_case`, `new_registration`, `reorganization`, `bankruptcy`, `liquidation`

## Правила расчёта веса

### 1. Базовый вес по типу события

| event_type | Базовый вес | Условие |
|---|---|---|
| `capital_change` | 6 | Рост капитала (увеличение, допэмиссия) |
| `capital_change` | 1 | Уменьшение капитала |
| `new_procurement` | 7 | — |
| `reorganization` | 5 | Слияние, присоединение, выделение |
| `new_registration` | 2 | — |
| `court_case` | 2 | Компания — истец |
| `court_case` | 1 | Компания — ответчик |
| `bankruptcy` | 1 | — |
| `liquidation` | 1 | — |

Различение `capital_change` рост vs уменьшение и `court_case` истец vs ответчик: анализировать поле `event_description`.

### 2. Модификатор суммы (amount в рублях)

| Диапазон | Модификатор |
|---|---|
| ≥ 500 000 000 (≥500М) | +3 |
| 100 000 000 – 499 999 999 (100–499М) | +2 |
| 50 000 000 – 99 999 999 (50–99М) | +1 |
| 10 000 000 – 49 999 999 (10–49М) | +0 |
| < 10 000 000 (<10М) | −1 |

### 3. Модификатор источника

| Источник | Модификатор |
|---|---|
| `e_disclosure` | +1 |
| `zakupki` | +1 |
| `kontur_zakupki` | +1 |
| `kad_arbitr` | −1 |
| Остальные | +0 |

### 4. Модификатор давности (от `detected_at` до текущего момента)

| Давность | Модификатор |
|---|---|
| < 24 часов | +1 |
| 1–3 дня | +0 |
| > 3 дней | −1 |

### 5. Финальный вес

```
финальный_вес = базовый + сумма_мод + источник_мод + давность_мод
финальный_вес = max(0, min(10, финальный_вес))
```

## Маппинг продуктов по типу события

| event_type | Рекомендованные продукты |
|---|---|
| `new_procurement` | Банковские гарантии, кредитование |
| `capital_change` (рост) | Кредитование, РКО |
| `capital_change` (уменьшение) | Факторинг |
| `reorganization` | Кредитование, РКО |
| `new_registration` | РКО, кредитование |
| `court_case` (истец) | Факторинг |
| `court_case` (ответчик) | — (не рекомендован для активных продаж) |
| `bankruptcy` | — (не рекомендован) |
| `liquidation` | — (не рекомендован) |

## Формат вывода

Markdown-таблица топ-20 событий (отсортированы по финальному весу убывание):

```
| # | Компания | Источник | Событие | Сумма | Вес | Продукты | Обоснование |
|---|----------|----------|---------|-------|-----|----------|-------------|
| 1 | ... | ... | ... | ... | ... | ... | ... |
```

- **Колонка «Событие»** — человекочитаемое название (не `event_type`), например «Новая закупка», «Увеличение капитала»
- **Колонка «Сумма»** — отформатированная: «150.0 млн ₽», «1.0 млрд ₽»
- **Колонка «Обоснование»** — краткое объяснение веса (1 фраза): какой триггер, какая сумма, почему релевантно для B2B
- После таблицы: примечание о компаниях 21+ (вес ≤ 2.0 — не входят в аналитику)
