# PROACT — B2B Pipeline для проактивного бизнес-обнаружения

Инструмент для автоматического выявления компаний-потенциальных клиентов банка по
публичным данным, оценки их рискового профиля, подготовки персонализированного
outreach и отслеживания воронки контактов.

Пайплайн из 5 последовательных стадий: **SCOUT → ANALYST → WARMUP → BRIEF → TRACKER**,
с замкнутым контуром обратной связи для самообучения.

## Что делает PROACT

1. **Сканирует** 10 публичных источников (госзакупки, суды, новости, реестры) на предмет
   корпоративных событий: новые регистрации, крупные контракты, допэмиссии, банкротства,
   судебные иски, найм, лидерские изменения.
2. **Оценивает** каждую компанию: риск-профиль, продуктовую матрицу (7 банковских продуктов),
   confidence-score, стратегию подхода.
3. **Готовит** персонализированные outreach-письма с привязкой к конкретному триггер-событию
   и продуктовому обоснованию (включая логику «ставка ниже конкурентов ↔ кросс-продажа»).
4. **Формирует** досье перед звонком/встречей: таймлайн событий, риск-факторы, план звонка,
   возражения и ответы, чек-лист подготовки.
5. **Отслеживает** воронку: кто контактирован, на какой стадии, какой результат.
6. **Учится** на обратной связи от менеджеров: корректирует продуктовые веса и тон писем.

## Архитектура

```
┌─────────────────────────────────────────────────────────────────┐
│                     pipeline_runner.py                          │
│                     (оркестратор)                               │
│                                                                 │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│  │  SCOUT   │──▶│ ANALYST  │──▶│  WARMUP  │──▶│  BRIEF   │──▶│ TRACKER  │
│  │ (сбор)   │   │ (оценка) │   │ (письма) │   │ (досье)  │   │ (воронка)│
│  └──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
│       │              │              │              │              │
│       ▼              ▼              ▼              ▼              ▼
│  scout_raw.json  analyst_*.json  warmup_*.md   briefs_json/   tracker_store
│                                                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                    Feedback Loop                             ││
│  │  feedback_store.json ──▶ feedback_applier ──▶ weights/tone  ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
```

**Два режима работы:**

| Режим | Описание | Стоимость |
|-------|----------|-----------|
| **Стандартный** (`python3 pipeline_runner.py`) | Все 5 стадий выполняются локально (Python-скрипты). Deterministic scoring, без LLM. | $0 |
| **Smart** (`--smart`) | SCOUT + TRACKER — локально; ANALYST, WARMUP, BRIEF — через LLM-субагентов для глубокого анализа и естественного текста. | LLM API |

## Быстрый старт

```bash
# Стандартный прогон (локально, все стадии)
python3 pipeline_runner.py

# Smart-режим (гибрид: локальный код + LLM-анализ)
python3 pipeline_runner.py --smart

# Smart + HTML-отчёты (общий + по-компанию)
python3 pipeline_runner.py --smart --file

# Применить накопленную обратную связь перед прогоном
python3 pipeline_runner.py --learn

# Записать результат контакта
python3 pipeline_runner.py --result "ПАО Аэрофлот" "связались"
```

## Режимы запуска

### Стандартный режим

Все стадии выполняются локальными Python-скриптами без вызова LLM:

```bash
python3 pipeline_runner.py [--top-n 50] [--format text|json] [--skip-warmup] [--threshold 0.9]
```

- SCOUT собирает события и сохраняет в `pipeline_store.json`
- ANALYST строит риск-профили и продуктовую матрицу
- WARMUP генерирует письма для компаний с confidence ≥ threshold
- BRIEF формирует досье для целевых компаний
- TRACKER выводит воронку контактов

### Smart-режим (`--smart`)

Гибридная архитектура: тяжёлые когнитивные стадии делегируются LLM, deterministic — остаются локальными.

**Фазы выполнения:**

1. `--smart` — SCOUT собирает сырые события локально, сохраняет `scout_raw.json`
2. Родительский LLM-агент анализирует данные, пишет `analyst_brief_llm.md` и `warmup_llm.md`
3. `--smart-collect` — санитайзинг, разбиение на файлы, TRACKER, HTML-генерация

```bash
# Шаг 1: SCOUT (локально)
python3 pipeline_runner.py --smart

# Шаг 2: LLM-анализ (выполняется оркестратором)
# ... агент читает scout_raw.json, пишет analyst_brief_llm.md + warmup_llm.md ...

# Шаг 3: Сборка + TRACKER + HTML
python3 pipeline_runner.py --smart-collect --file
```

### Режим `--file` (HTML-отчёты)

Генерирует два типа HTML-отчётов:

- **Общий отчёт** (`reports/report_YYYY-MM-DD.html`) — все 5 стадий на одной странице
  с навигацией, тёмной темой, Markdown→HTML конверсией
- **По-компанию брифы** (`reports/briefs/Название.html`) — интерактивные досье с
  кольцом скоринга, таймлайном, риск-факторами, продуктовой матрицей, планом звонка

## Компоненты

### SCOUT — сбор событий

**Файлы:** `scout_pipeline.py`, `data/skills/external/scout/`

Собирает корпоративные события из 10 источников, дедуплицирует, нормализует в единую схему:

```json
{
  "id": "a3f8b2c1",
  "company_name": "ПАО Роснефть",
  "inn": "7706648991",
  "source": "e_disclosure",
  "event_type": "share_consolidation",
  "event_description": "Консолидация акций на 500 млн ₽",
  "amount": 500000000,
  "detected_at": "2026-07-17T09:00:00Z"
}
```

**Вес события** (рассчитывается локально): `recency × probability × margin`

Шкала 0–10, где:
- 9–10: крупные капитальные операции (допэмиссии, консолидация ≥ 300М ₽)
- 7–8: крупные контракты, расширение производства
- 5–6: новости о росте, новые магазины/филиалы
- 1–3: банкротства, суды (намеренно низкий приоритет — это риски, не возможности)

**Бизнес-логика скоринга:** банкротства и судебные иски — это сигналы риска, а не
возможности для продажи. Они получают низкий вес, чтобы не загромождать топ лидов.

### ANALYST — риск-профилирование

**Файл:** `analyst_pipeline.py`, `analyst_output.py`

Для каждой компании из SCOUT строит:

| Поле | Описание |
|------|----------|
| `risk_profile` | Уровень риска: `very_low` → `very_high` (на основе судов, банкротств, фин. метрик) |
| `product_heatmap` | Матрица 7 продуктов × confidence-score (0.0–1.0) |
| `entry_scenario` | Стратегия подхода: `warm_lead`, `approach_carefully`, `nurture`, `avoid` |
| `overall_score` | Сводный показатель привлекательности |

**Продуктовая матрица (7 продуктов):**

| Код | Продукт | Когда релевантен |
|-----|---------|-------------------|
| `rko` | Расчётно-кассовое обслуживание | Все компании (базовый) |
| `lending` | Кредитование | Контракты, капвложения, оборотный капитал |
| `bank_guarantee` | Банковские гарантии | Госзаказ (44-ФЗ, 223-ФЗ), обеспечение контрактов |
| `leasing` | Лизинг | Закупка оборудования, транспорта, спецтехники |
| `salary_project` | Зарплатный проект | 500+ сотрудников |
| `ved` | ВЭД / валютный контроль | Экспорт/импорт, валютные операции |
| `acquiring` | Эквайринг | Розница, e-commerce, платёжные терминалы |

В Smart-режиме ANALYST выполняется LLM-субагентом, который читает `scout_raw.json`
и пишет развёрнутые профили с обоснованием, стратегией по сегментам и конкретными
точками входа.

### WARMUP — outreach письма

**Файл:** `warmup_pipeline.py`, `prompts/warmup.md`

Генерирует персонализированные email-шаблоны для компаний с confidence ≥ порога
(по умолчанию 90%).

**Структура письма:**
- Тема письма (с продуктом и конкретным оффером)
- Контекст: ссылка на триггер-событие из SCOUT
- Оффер: конкретный продукт со ставкой/условиями
- Обоснование: почему именно этот продукт, почему сейчас
- Призыв к действию

**Ключевая бизнес-логика — «Ставка ниже конкурентов»:**

> Снижение ставки относительно конкурентов возможно максимум на 1–3%.
> Потерянная маржа компенсируется кросс-продажей (РКО, эквайринг, зарплатный проект,
> ВЭД, факторинг, лизинг). Без кросс-продукта ставку не упоминать — скидка без
> компенсации = подарок, не бизнес-решение.

Тон писем адаптируется через feedback-контур: `professional` (по умолчанию) или `casual`.

---

### BRIEF — досье перед встречей

**Файлы:** `brief_pipeline.py`, `brief_html_generator.py`, `prompts/brief.md`, `prompts/brief_json.md`

Формирует структурированное досье по компании перед звонком или встречей.

**Markdown-досье включает:**
- Профиль компании (ИНН, отрасль, регион)
- Точка входа с контекстом триггера
- Риск-факторы (ok / warn / bad)
- Финансовая динамика (2 периода + дельта)
- Продуктовая матрица: основной продукт + кросс-селл + дополнительно
- План звонка (timeline 3–5 шагов)
- Вопросы для уточнения
- Возражения и ответы
- Чек-лист подготовки

**HTML-досье** (`brief_html_generator.py`) рендерит JSON в интерактивную страницу:

| Секция | Описание |
|--------|----------|
| Hero | Название, ИНН, отрасль, score-кольцо, чипы-индикаторы |
| Entry banner | Точка входа с контекстом триггер-события |
| Контакты | Контактные лица с заметками |
| Таймлайн | Хронология событий |
| Риск-факторы | Цветовая индикация (ok/warn/bad) |
| Финансы | 2 периода + дельта с цветовым кодом |
| Продукты | Основной + кросс-селл с обоснованием |
| Связи | Корпоративные связи компании |
| План звонка | Timeline с шагами |
| Возражения | Таблица «возражение → ответ» |
| Чек-лист | Подготовка к встрече |

JSON-схема для HTML-генерации описана в `prompts/brief_json.md`.

### TRACKER — воронка контактов

**Файл:** `tracker_pipeline.py`, `tracker_store.json`

Журнал контактов и аналитика воронки:

| Компания | Статус | Дата | Канал |
|----------|--------|------|-------|
| ПАО Аэрофлот | ⏳ Ожидаем дальнейших касаний | 16.07.2026 | звонок |
| ПАО Роснефть | 📧 Письмо отправлено | 17.07.2026 | email |

**Статусы:**
- `📧 Письмо отправлено` — WARMUP-письмо отправлено
- `⏳ Ожидаем дальнейших касаний` — первый контакт состоялся, ждём развития
- `📅 Встреча назначена` — согласована встреча
- `✅ Сделка закрыта` — успешная конверсия
- `❌ Отказ` — клиент отказал

TRACKER также отслеживает, какие компании уже были в WARMUP, чтобы избежать
дублирования outreach.

### Feedback — контур самообучения

**Файлы:** `feedback_pipeline.py`, `feedback_applier.py`, `feedback_store.json`, `feedback_adjustments.json`

Замкнутый контур: менеджер оставляет обратную связь → система корректирует веса и тон.

**Сбор обратной связи:**

```bash
# Структурированная: компания + продукт + результат
python3 pipeline_runner.py --feedback-record "ПАО Аэрофлот" "lending" "клиент отказал: ставка не конкурентная"

# Свободная форма (парсится на сигналы)
python3 pipeline_runner.py --feedback-general "письма слишком длинные, ставку указывай точнее"
```

**Классификация сигналов:**
- Позитивные: продукт сработал → вес продукта повышается (+0.10)
- Негативные: продукт не сработал → вес понижается (−0.15)
- Тон: «слишком формально» → casual, «слишком просто» → formal
- Суды: «судебное дело отпугнуло» → penalty за суды увеличивается

**Применение корректировок:**

```bash
# Автоматически (после каждого прогона — последние 24 часа)
python3 pipeline_runner.py

# Вручную (вся накопленная обратная связь)
python3 pipeline_runner.py --learn

# Посмотреть текущие корректировки
python3 feedback_applier.py --status

# Сбросить к значениям по умолчанию
python3 feedback_applier.py --reset
```

**Границы корректировок:**
- Вес продукта: 0.2 – 2.0 (от базового 1.0)
- Penalty за суды: 0.05 – 0.8
- Тон: `professional` / `casual` / `formal`

### HTML-генераторы отчётов

**`html_generator.py`** — общий отчёт

Объединяет 5 markdown-файлов (scout/analyst/warmup/brief/tracker) в одну HTML-страницу.
Тёмная тема, sticky-навигация, Markdown→HTML конверсия (таблицы, заголовки, списки,
цитаты, код). Сохраняется в `reports/report_YYYY-MM-DD.html`.

**`brief_html_generator.py`** — по-компанию брифы

Принимает JSON-файл со структурированными данными компании, рендерит в интерактивную
HTML-страницу по шаблону PROACT. Полноценная CSS-тема (переменные, состояния, адаптив).

```bash
# Один файл
python3 brief_html_generator.py reports/briefs_json/company.json

# Все JSON в директории
python3 brief_html_generator.py --dir reports/briefs_json/
```

**`sanitize_llm_output.py`** — очистка LLM-вывода

Удаляет мета-комментарии, технический мусор, code-block artifacts из вывода
LLM-субагентов перед доставкой в чат или записи в файлы. Используется в `--smart-collect`.

## Промпт-шаблоны

LLM-промпты для Smart-режима хранятся в `prompts/`:

| Файл | Назначение | Используется в |
|------|------------|----------------|
| `scout_scorer.md` | Промпт для LLM-скоринга сырых событий SCOUT | Smart: анализ scout_raw.json |
| `analyst.md` | Промпт для риск-профилирования и продуктовой матрицы | Smart: ANALYST-стадия |
| `warmup.md` | Промпт для генерации outreach-писем | Smart: WARMUP-стадия |
| `brief.md` | Промпт для текстового досье перед встречей | Smart: BRIEF-стадия |
| `brief_json.md` | Промпт для JSON-структуры под HTML-генерацию | Smart: BRIEF + `--file` |

**Бизнес-правила в промптах:**
- Ставка ниже конкурентов: max 1–3% снижение, компенсация кросс-продажей
- Для высоколиквидных компаний (после допэмиссий): не предлагать «сохранение ликвидности» —
  использовать опцион, скорость, сценарную полезность
- Кросс-селл с двойным обоснованием: (а) зачем продуктом компании, (б) почему он делает
  ставку ниже конкурентов экономически возможной
- Банкротства и суды — не推销ать, а учитывать в риск-профиле

## CLI-справочник

### Основные команды

```bash
# Полный прогон (стандартный режим)
python3 pipeline_runner.py

# Smart-режим (гибрид LLM + локальный код)
python3 pipeline_runner.py --smart
python3 pipeline_runner.py --smart-collect --file

# Одиночные стадии (для интеграции)
python3 pipeline_runner.py --stage scout
python3 pipeline_runner.py --stage tracker
```

### Флаги

| Флаг | Описание | По умолчанию |
|------|----------|--------------|
| `--top-n N` | Количество строк в таблицах | 50 |
| `--format text\|json` | Формат вывода | text |
| `--limit N` | Макс. событий от одного источника | 1000 |
| `--extra-info` | Включить сводки сканирования | off |
| `--dry-run` | Не записывать в store | off |
| `--skip-warmup` | Пропустить стадию WARMUP | off |
| `--skip-tracker` | Пропустить стадию TRACKER | off |
| `--threshold F` | Порог confidence для WARMUP | 0.9 (90%) |
| `--brief COMPANY` | Досье для конкретной компании | — |
| `--smart` | Smart-режим (SCOUT локально → LLM) | off |
| `--smart-collect` | Сборка smart-результатов + TRACKER | off |
| `--file` | Генерация HTML-отчётов | off |

### Обратная связь

```bash
# Записать результат контакта
python3 pipeline_runner.py --result "Компания" "статус"

# Структурированная обратная связь
python3 pipeline_runner.py --feedback-record "Компания" "Продукт" "Результат"

# Свободная форма
python3 pipeline_runner.py --feedback-general "текст обратной связи"

# Статистика обратной связи
python3 pipeline_runner.py --feedback-stats

# Список всех записей
python3 pipeline_runner.py --feedback-list

# Применить обратную связь (обновить веса и тон)
python3 pipeline_runner.py --learn

# Сбросить корректировки
python3 feedback_applier.py --reset
```
